# p4y 설계 문서 (Design Specification)

**p4y**는 Perforce Visual Client(P4V)의 핵심 기능을 Rust와 Iced 프레임워크로 재구현한 가볍고 빠른 읽기 전용 워크스페이스 브라우저입니다.

## 1. 개요 (Overview)
- **목적:** 실제 로컬 파일 동기화 없이 Perforce 서버의 워크스페이스 매핑을 해석하여 파일 구조를 탐색하고, 히스토리(Changelist) 및 파일 내용을 신속하게 조회함.
- **핵심 가치:** 
  - **Lightweight:** P4V보다 가벼운 리소스 사용.
  - **Read-only Focus:** 실수로 파일을 수정하거나 체크아웃할 염려 없는 안전한 브라우징.
  - **Timeline-centric:** 파일의 변경 이력(CL)을 파악하는 데 최적화된 UX.

## 2. 사용자 인터페이스 (User Interface)

### 2.1 레이아웃 구조
- **좌측 (Navigation Tree):** 가상 워크스페이스 트리. `p4 dirs` 및 `p4 files`를 이용한 지연 로딩(Lazy Loading).
- **중앙 (History List):** 선택된 파일/폴더의 리비전 목록 (Changelist Feed).
- **우측 (CL Details):** 선택된 CL의 상세 정보 (Description, Affected Files).
- **모달 (Content Viewer):** 파일의 실제 내용 조회 (별도 팝업).

### 2.2 워크플로우
1. 사용자가 좌측 트리에서 파일을 선택.
2. 우측 상단에 해당 파일의 히스토리 업데이트.
3. 히스토리에서 특정 CL 선택 시 우측 하단에 상세 정보 표시.
4. 상세 정보의 파일 목록에서 [내용 보기] 클릭 시 모달로 본문 출력.

## 3. 기술 아키텍처 (Technical Architecture)

### 3.1 기술 스택
- **언어:** Rust
- **GUI 프레임워크:** Iced (Elm Architecture 기반)
- **P4 연동:** 시스템 `p4` CLI 호출 (Asynchronous)
- **상태 관리:** Iced의 Message/Update 패턴 활용

### 3.2 P4 연동 전략
- **명령어 실행:** `std::process::Command`를 비동기(`tokio` 또는 `iced::Subscription`)로 실행.
- **데이터 포맷:** `-Ztag` 옵션을 사용하여 파싱 효율 극대화.
- **가상 트리 구현:** 
  - `p4 dirs //workspace/path/*`로 하위 폴더 조회.
  - `p4 files //workspace/path/*`로 하위 파일 조회.
- **성능 최적화 (Caching):**
  - **Metadata Cache:** 한 번 가져온 CL 상세 정보는 메모리에 유지.
  - **Tree Cache:** 이미 확장된 트리 노드의 정보는 유지하고 [새로고침] 시에만 갱신.

## 4. 데이터 모델 (Data Model)

```rust
struct AppState {
    workspace: String,
    tree_root: TreeNode,
    selected_path: Option<String>,
    history: Vec<Changelist>,
    selected_cl: Option<ChangelistDetail>,
    content_modal: Option<FileContent>,
}

struct Changelist {
    id: u32,
    author: String,
    date: DateTime,
    desc_short: String,
}

struct ChangelistDetail {
    id: u32,
    description: String,
    affected_files: Vec<AffectedFile>,
}
```

## 5. 단계별 구현 계획 (Phases)

### Phase 1: 기반 구조 구축
- P4 접속 설정 및 인증 로직.
- 좌측 워크스페이스 트리 레이아웃 및 기본 지연 로딩 구현.

### Phase 2: 히스토리 인스펙터
- 파일 선택 시 히스토리(CL 목록) 조회 및 표시.
- CL 선택 시 상세 정보(Description, Affected Files) 조회.

### Phase 3: 콘텐츠 뷰어 및 고도화
- 파일 내용 조회 모달 구현.
- 데이터 캐싱 로직 및 새로고침 기능.
- UI/UX 폴리싱 (Iced 스타일링).

## 6. 자체 검토 (Self-Review)
- **모호성:** "로컬 파일 연동 없음"을 명확히 함 (가상 트리 방식).
- **일관성:** 트리와 히스토리 뷰의 관계가 명확히 정의됨.
- **확장성:** 추후 Diff 기능이나 Syntax Highlighting 추가가 용이한 구조임.
